## 什么是 MCP

MCP（Model Context Protocol）是 Anthropic 推出的开放协议，用于标准化 AI 模型与外部工具、数据源之间的通信。

### 核心设计目标

- **标准化接口**：统一 AI 与外部系统的通信方式
- **安全隔离**：通过本地服务器运行，保护敏感数据
- **可扩展性**：任何开发者都可以创建 MCP 服务器
- **互操作性**：不同 AI 应用可以共享相同的工具集

### MCP 架构组成

```
┌────────────────────────────────────────┐
│              AI 模型 (Claude)           │
│         ┌─────────────────┐            │
│         │   MCP Client    │            │
│         │  (内置于 Claude Code)        │
│         └────────┬────────┘            │
└──────────────────┼─────────────────────┘
                   │ JSON-RPC 2.0
                   ▼
┌────────────────────────────────────────┐
│           MCP Server (本地/远程)        │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐  │
│  │ 工具调用 │ │ 文件读取 │ │ API访问 │  │
│  └─────────┘ └─────────┘ └─────────┘  │
└────────────────────────────────────────┘
```

## 什么是 SKILLS

SKILLS 是基于 MCP 协议封装的具体能力单元，可以理解为 "AI 的专业技能包"。

### SKILLS 的本质

- **能力封装**：将特定领域的能力打包成可调用的模块
- **上下文感知**：可以理解并适应不同的使用场景
- **可组合性**：多个 SKILLS 可以协同工作

### 常见 SKILLS 类型

| 类型 | 示例 | 功能描述 |
|------|------|----------|
| **浏览器自动化** | playwright-skill | 截图、点击、表单填写 |
| **代码操作** | git-master | commit、rebase、分支管理 |
| **UI设计** | frontend-ui-ux | 生成组件、优化界面 |
| **文档处理** | openspec-* | PRD分析、代码生成 |
| **数据分析** | data-analyzer | 处理 CSV、生成图表 |

```
        │
        ▼ 返回结果

│   AI 模型      │── 整合结果，生成回复
└───────────────┘
```

## 最佳实践

### 1. 选择合适的 Granularity

- **粗粒度**：一个 SKILL 完成一个完整任务（如"部署应用到 AWS"）
- **细粒度**：多个小 SKILL 组合完成复杂任务（如"上传文件"+"配置域名"+"启动服务"）

### 2. 错误处理与重试

```python
@mcp.tool()
def reliable_operation():
    """带重试机制的可靠操作"""
    for attempt in range(3):
        try:
            return do_operation()
        except TransientError:
            if attempt == 2:
                raise
            time.sleep(2 ** attempt)  # 指数退避
```

### 3. 安全考虑

- 敏感操作需要用户确认
- 限制文件系统访问范围
- 审计日志记录
