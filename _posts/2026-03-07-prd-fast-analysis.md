# PRD 快速分析

> 在 PRD 评审会议前进行的快速分析，目的是识别 PRD 中的模糊点、矛盾点和风险点，带着问题去开会

---

## 目的

- **识别问题**：找出 PRD 中描述不清晰、有歧义的地方
- **风险评估**：识别技术风险和依赖关系
- **提高效率**：带着问题去开会，避免会上浪费时间

---

## 核心动作

### 1. 文档结构扫描（10 分钟）

快速浏览 PRD 目录，了解：
- 有哪些功能模块
- 每个模块的页码/位置
- 文档的整体结构

### 2. 模块化梳理（30-60 分钟）

对每个功能模块：
- 用一句话描述模块功能
- 列出涉及的角色
- 列出核心功能点（3-5 个）
- 初步识别关键字段（5-10 个）
- 标记可疑点/需确认项

### 3. CheckList 检查（30 分钟）

对照 PRD 分析 CheckList 快速过一遍：
- 字段完整性
- 按钮完整性
- 业务逻辑
- API 接口
- 异常场景

### 4. 问题清单整理（30 分钟）

将不确定项整理为问题清单：
- 高优先级问题（必须在评审会上确认）
- 中优先级问题（有时间则讨论）
- 低优先级问题（可会后沟通）

## 技巧

### 如何快速识别问题

1. **找绝对词**：「必须」、「只能」、「所有」——这些地方容易有遗漏
2. **找模糊词**：「等」、「相关」、「适当」——这些需要明确
3. **找跨模块**：涉及多个模块交互的地方容易有接口问题
4. **找状态变更**：状态流转复杂的地方容易有逻辑漏洞

### 如何提问

- **具体化**：不要问「这个清楚吗？」，要问「这个字段的校验规则是什么？」
- **给选项**：「这个字段是必填还是选填？」而不是「这个字段有什么要求？」
- **讲场景**：「如果用户这样操作，会发生什么？」
